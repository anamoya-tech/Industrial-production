## 🤖 Uso de IA en el proyecto

La inteligencia artificial se utilizó como herramienta de apoyo en distintas fases del proyecto, siguiendo un enfoque práctico y orientado a productividad.

### 📂 Generación de datos

Se utilizó **Claude (IA)** para generar el dataset inicial a partir de una estructura previamente definida.

- Se diseñaron las variables manualmente (contexto industrial)
- Se solicitó la generación de un dataset **intencionadamente sucio**
- Se incluyeron errores reales:
  - valores nulos
  - formatos inconsistentes
  - errores tipográficos
  - incoherencias en datos numéricos

👉 Objetivo: simular un entorno real de trabajo con datos imperfectos

---

### 🧹 Apoyo en limpieza y SQL

La IA se utilizó como soporte para:

- plantear estrategias de limpieza
- validar enfoques de transformación
- optimizar queries SQL

👉 Siempre con criterio propio:
- entendiendo cada transformación
- adaptando las soluciones al dataset

---

### 📊 Apoyo en análisis

Se utilizó IA para:

- estructurar el análisis exploratorio (EDA)
- identificar posibles enfoques analíticos
- mejorar la interpretación de resultados

👉 Enfocado a:
- obtener insights de negocio
- no solo resultados técnicos

---

### 🧠 Enfoque aplicado

La IA se utilizó como **herramienta de apoyo**, no como sustituto del análisis.

El trabajo se centró en:

- comprender los datos
- tomar decisiones de limpieza
- interpretar resultados
- extraer conclusiones útiles

---

### 🚀 Valor añadido

El uso de IA permitió:

- acelerar el desarrollo del proyecto
- trabajar con un dataset más realista
- mejorar la calidad del análisis
- enfocarse en la parte analítica y de negocio

---

💡 Este enfoque refleja el uso actual de IA en entornos profesionales de datos, donde actúa como un acelerador del trabajo del analista.
